# i18n: every user-facing string through the translation layer

**Rule.** A user-facing string that bypasses the i18n layer is invisible debt: it
ships untranslated and nobody notices until a user in the other language hits it.
Locale files stay key-for-key identical; completeness must be diffable, not
discovered.

## Rules

1. **Everything user-facing goes through `t()`** — visible text, `title=`
   tooltips, `placeholder=`, button labels, option labels and hints, status
   lines. The only exemptions are developer-facing debug output, log events,
   and **language-neutral example literals** (a placeholder showing a URL
   shape, a dimension like `1024x1024`, a bare number) — translating those is
   noise. A placeholder containing any prose word is not exempt. The exemption
   list is deliberately tiny so the rule stays mechanical.
2. **Keys are `scope.key`**, scope = the owning feature (`provider.title`,
   `workspace.modeAsk`). A key's scope tells the reader where it renders.
3. **Key parity is an invariant, not a goal.** Every locale file holds exactly
   the same key set; adding a key to one file without the others is a defect.
   Untranslated fallback text in a secondary locale is an acceptable stopgap —
   a missing key is not. Parity is checked by diffing key sets, which makes
   translation completeness a one-liner instead of a manual audit.
4. **Constants store keys, not strings.** A module-level option list (labels,
   hints, modes) can't call `t()` at definition time — it stores i18n keys
   (`labelKey`, `hintKey`) and translates at render. Never hardcode the label
   into the constant "because it's just three words".
5. **Plurals and interpolation use the library's conventions** (`_other`
   suffixes, `{{var}}` slots) — never string-concatenate around a translated
   fragment, which breaks word order in other languages.
6. **Markup inside a translated sentence** goes through the library's component
   interpolation (e.g. `Trans` with a components map), not by splitting the
   sentence into fragments — fragments translate wrong.

## Why

Hardcoded strings are greppable only if you know to grep; key parity is
diffable by anyone. Both properties turn "is the app translated?" from a QA
pass in every language into a mechanical check.
